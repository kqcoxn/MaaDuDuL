## 打开并配置模拟器

在一切准备就绪后，**不要着急打开 MDDL，请优先打开模拟器。**

虽然 MaaFramework 框架支持[大多数模拟器](https://maa.plus/docs/zh-cn/manual/device/)，但 MDDL **仅推荐使用 [mumu](https://mumu.163.com/) 模拟器作为游戏载体。MDDL 在测试时使用 mumu12-v5 官方版**。

在打开模拟器后，请在模拟器设置界面正确设置分辨率。**模拟器分辨率应为 `16:9` 比例（mumu12 的平板预设均为此比例），符合比例的分辨率无大小要求，最低帧率为 `30hz`**（包括自动降帧）。

**请确保模拟器及相关配置符合要求，否则报错将不会被解决。**

## 启动 MDDL

在配置好模拟器后，返回解压的文件夹，可以在文件夹内找到 `MaaDuDuL.exe`（部分环境下无后缀）：

![](/images/user/MDDL执行文件.png)

请**右键`MDDL.exe`后选择以管理员方式运行**

::: details 为什么任务栏的图标是牛牛？
MDDL 使用 MFAAvalonia 作为 GUI。MFAAvalonia 是 MaaFw 系列的 GUI 开源项目，因为 MaaFw 最早是由方舟的 Maa 演化而来，因此 MFAAvalonia 的作者选择类似于 Maa 的牛牛图标作为 icon，也因此牛牛图标会被保留。
:::

::: tip
您可以右键 `MaaDuDuL.exe`，并依次选择`(win11:显示更多选项)`->`发送到`->`桌面快捷方式`在桌面创建一个快捷方式，方便以后在桌面直接启动。
:::

## 选择更新源

在打开 MDDL 后，**您可能会看到弹出的检查更新失败提示，可以先忽略**，并选择合适的更新源。

您可以在 Github 与 Mirror 酱两种更新源中选择，首先，打开设置界面，并下划至更新设置部分：

![](/images/user/更新设置.png)

在资源下载源配置中，选择 `Github` 或 `Mirror 酱`

MDDL 的默认下载源为`Mirror 酱`，**若您没有 Mirror 酱服务，请选择 `Github`作为更新源**。

::: tip 可能的问题：Github 源检查更新失败/更新缓慢

国内对于 Github 的防火墙时有时无，且速度非常不稳定。如果条件允许，请使用网络代理或加速器访问。

Github 本身的下载速度很快，如果您无法通过其他途径优化下载链路，可以使用 Mirror 酱或手动重新下载（可覆盖更新，配置会保留）的方式更新。
:::

## 连接模拟器

点击左侧任务图标回到主界面，MDDL 需要连接模拟器，以获取模拟器 ADB 支持。

第一次连接模拟器或运行任务时，Agent 将会调用程序自带的 Python 环境与 pip 下载 MDDL 所需的动态依赖，因此等待时间可能较久（应不超过 5 分钟），请耐心等待。

面板右侧日志将会提示目前的状态：

![](/images/user/init.png)

如果提示任务已全部完成，恭喜您已经可以愉快的使用 MDDL 了！
