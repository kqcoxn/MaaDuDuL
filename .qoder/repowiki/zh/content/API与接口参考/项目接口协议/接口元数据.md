## 简介

ProjectInterface V2协议是MaaFramework的一个标准化项目结构声明，通过`interface.json`文件定义项目的元数据信息。这些元数据不仅为项目提供了基本的标识信息，还直接影响用户界面的显示、版本管理和资源分发。在MaaDuDuL项目中，这些元数据字段被精心配置，以提供清晰的用户指引和高效的自动化任务管理。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#简介)

## 核心元数据字段详解

### interface_version

`interface_version`字段用于标识ProjectInterface协议的版本号。在当前协议中，该值固定为2，表示使用的是ProjectInterface V2协议。这个版本号是强制性的，必须正确设置，以便客户端工具能够识别和处理项目配置。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#整体结构)

### name

`name`字段是项目的唯一标识符，用作项目ID。在MaaDuDuL项目中，该值设置为"MaaDuDuL"，作为项目的核心标识。这个名称在系统内部被广泛引用，是区分不同项目的基础。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#整体结构)

### label

`label`字段是项目的显示名称，用于在用户界面中展示。如果未设置，将默认显示`name`字段的值。该字段支持国际化字符串（以`$`开头），允许在不同语言环境下显示相应的本地化名称。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#整体结构)

### title

`title`字段定义了窗口标题，客户端会直接显示该内容而不添加其他修饰。在MaaDuDuL项目中，标题包含了版本信息和游戏名称，为用户提供清晰的上下文。如果未设置，将默认使用`name`和`version`拼接生成。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#整体结构)

### icon

`icon`字段指定了应用图标文件的路径，相对于项目根目录。在MaaDuDuL项目中，图标文件为"logo.png"。如果未指定该字段，将使用默认图标。该字段同样支持国际化。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#整体结构)

### version

`version`字段表示项目版本号，客户端可以将其展示给用户，并用于版本更新检查。在MaaDuDuL项目中，当前版本为"v0.1.2"，遵循特定的版本格式说明，包含迭代版本、大版本和小版本。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#整体结构)
- [90.更新记录.md](file://assets/resource/Announcement/90.更新记录.md#版本格式说明)

### description

`description`字段提供了项目描述信息，通常显示在"关于"页面。该字段支持文件路径、URL或直接文本，内容支持Markdown格式。在MaaDuDuL项目中，描述为"嘟嘟脸小助手"，简洁明了地传达了项目的核心功能。

**文档来源**
- [3.3-ProjectInterfaceV2协议.md](file://instructions/maafw-guide/3.3-ProjectInterfaceV2协议.md#整体结构)

### welcome

`welcome`字段定义了欢迎消息，在用户首次使用时弹窗显示，也可作为公告使用。系统会记录显示内容，当内容更新时会再次弹窗显示。在MaaDuDuL项目中，欢迎消息为"欢迎使用 MDDL!"，为新用户提供友好的入门指引。

### contact

`contact`字段包含了联系方式信息，显示在"关于"页面。支持文件路径、URL或直接文本，内容支持Markdown格式。在MaaDuDuL项目中，联系方式为"QQ群: 926874985"，为用户提供了一个直接的交流渠道。

### license

`license`字段指定了项目许可证信息，显示在"关于"页面。支持文件路径、URL或直接文本，内容支持Markdown格式。在MaaDuDuL项目中，许可证信息指向"LICENSE"文件，明确了项目的开源许可条款。

`github`字段包含了项目GitHub仓库地址，用于版本更新检查和问题反馈。在MaaDuDuL项目中，该值为"https://github.com/kqcoxn/MaaDuDuL"，连接了项目的开源社区和开发动态。

## 元数据对用户界面和版本管理的影响

接口元数据在MaaDuDuL项目中对用户界面显示和版本管理产生了重要影响。`title`字段直接决定了应用程序窗口的标题，为用户提供了明确的上下文信息。`icon`字段确保了应用程序在任务栏和桌面显示正确的图标，增强了品牌识别度。

在版本管理方面，`version`和`github`字段共同实现了自动更新检查功能。客户端工具可以通过GitHub API检查最新版本，并提示用户更新。`welcome`字段的版本感知特性确保了当项目发布新公告时，所有用户都能及时收到通知。

此外，`mirrorchyan_rid`和`mirrorchyan_multiplatform`等扩展字段支持了MirrorChyan资源分发平台，实现了国内高速下载和更新服务，极大地改善了用户的下载体验。

## 多语言环境处理

ProjectInterface V2协议通过`languages`字段支持多语言配置。虽然MaaDuDuL项目当前主要使用中文，但协议支持通过定义不同语言代码对应的翻译文件来实现国际化。

在`interface.json`中，可以通过`languages`对象指定不同语言的翻译文件路径，例如：
```json
"languages": {
    "zh_cn": "interface_zh.json",
    "en_us": "interface_en.json"
}
```

对于所有支持国际化的字符串字段，如果字符串以`$`开头，则表示该字符串是国际化字符串，客户端需要从相应的翻译文件中读取实际值再显示。这种机制允许项目在不修改主配置文件的情况下支持多种语言。
